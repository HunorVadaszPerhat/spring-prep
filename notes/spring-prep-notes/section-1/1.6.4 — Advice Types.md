# Objective 1.6.4 — Advice Types

## The five advice types

Each advice type defines *when* the advice runs relative to the matched method execution. The exam tests which type fires when, what parameters each receives, and the critical `@Around` requirement.

---

## `@Before`

Runs before the target method executes. Cannot prevent the method from running (unless it throws an exception).

```java
@Aspect
@Component
public class LoggingAspect {

    @Before("execution(* com.example.service.*.*(..))")
    public void logBefore(JoinPoint joinPoint) {
        System.out.println("About to call: " + joinPoint.getSignature().getName());
        // target method runs after this returns
        // throwing an exception here WOULD prevent the method from running
    }
}
```

Return type must be `void` — there's no return value to influence here.

---

## `@After`

Runs after the target method completes — whether it returned normally or threw an exception. Equivalent to a `finally` block.

```java
@After("execution(* com.example.service.*.*(..))")
public void logAfter(JoinPoint joinPoint) {
    System.out.println("Finished: " + joinPoint.getSignature().getName());
    // runs regardless of whether method succeeded or threw
}
```

⚠️ **EXAM TRAP**: `@After` runs in all cases — success and exception. It does **not** receive the return value or the exception. If you need either of those, use `@AfterReturning` or `@AfterThrowing`.

---

## `@AfterReturning`

Runs only when the target method returns normally — not if it throws. Receives the return value.

```java
@AfterReturning(
    pointcut = "execution(* com.example.service.OrderService.findOrder(..))",
    returning = "result"  // binds return value to this parameter name
)
public void logReturnValue(JoinPoint joinPoint, Object result) {
    System.out.println("Returned: " + result);
}
```

The `returning` attribute name must match the parameter name exactly. The parameter type acts as a filter — if you declare `Order result` instead of `Object result`, the advice only fires when the return type is assignable to `Order`.

⚠️ **EXAM TRAP**: `@AfterReturning` cannot modify the return value. You receive it for inspection only. To modify the return value, you need `@Around`.

---

## `@AfterThrowing`

Runs only when the target method throws an exception — not if it returns normally. Receives the exception.

```java
@AfterThrowing(
    pointcut = "execution(* com.example.service.*.*(..))",
    throwing = "ex"  // binds exception to this parameter name
)
public void logException(JoinPoint joinPoint, Exception ex) {
    System.out.println("Exception in " + joinPoint.getSignature().getName() 
                       + ": " + ex.getMessage());
}
```

Same binding rules as `@AfterReturning` — parameter name must match the `throwing` attribute, and the parameter type acts as a filter.

⚠️ **EXAM TRAP**: `@AfterThrowing` does **not** suppress the exception. The exception still propagates after the advice runs. To suppress or replace an exception, you need `@Around`.

---

## `@Around`

The most powerful advice type. Wraps the entire method execution — runs before and after, controls whether the method executes at all, can modify arguments, modify the return value, and suppress or replace exceptions.

```java
@Around("execution(* com.example.service.*.*(..))")
public Object timeExecution(ProceedingJoinPoint joinPoint) throws Throwable {
    long start = System.currentTimeMillis();

    Object result = joinPoint.proceed(); // target method executes HERE
                                         // must call this or method never runs

    long elapsed = System.currentTimeMillis() - start;
    System.out.println(joinPoint.getSignature().getName() + " took " + elapsed + "ms");

    return result; // must return the result (or a replacement)
}
```

Note `ProceedingJoinPoint` — a subtype of `JoinPoint` that adds `proceed()`. Only `@Around` uses this type.

⚠️ **EXAM TRAP**: If you forget to call `joinPoint.proceed()`, the target method **never executes**. No error is thrown — the method is silently skipped. This is one of the most dangerous mistakes in AOP code and a guaranteed exam question.

⚠️ **EXAM TRAP**: `@Around` advice must declare `throws Throwable` because `proceed()` can throw anything. The return type must be `Object` (or a compatible type) — you're responsible for returning the result.

---

## Modifying return value and suppressing exceptions with `@Around`

```java
@Around("execution(* com.example.service.OrderService.findOrder(..))")
public Object wrapResult(ProceedingJoinPoint joinPoint) throws Throwable {
    try {
        Object result = joinPoint.proceed();
        // modify or replace the return value
        return enrichWithMetadata(result);
    } catch (OrderNotFoundException ex) {
        // suppress the exception and return a default instead
        return Order.empty();
    }
}
```

This is something no other advice type can do — only `@Around` can modify the return value or suppress exceptions.

---

## Passing modified arguments to `proceed()`

```java
@Around("execution(* com.example.service.*.*(..)) && args(orderId)")
public Object validateAndProceed(ProceedingJoinPoint joinPoint, Long orderId) 
        throws Throwable {
    if (orderId <= 0) {
        throw new IllegalArgumentException("Invalid order ID");
    }
    return joinPoint.proceed(new Object[]{orderId}); // pass (possibly modified) args
}
```

`proceed()` with no arguments uses the original arguments. `proceed(args)` lets you replace them.

---

## Execution order when multiple advice applies

When multiple advice methods match the same join point, the order within one aspect follows advice type precedence. Across multiple aspects, use `@Order`:

```java
@Aspect
@Component
@Order(1)  // lower number = higher priority = outermost in the call stack
public class SecurityAspect { }

@Aspect
@Component
@Order(2)
public class LoggingAspect { }
```

For a single method call with both aspects:
- Security advice `@Before` fires first
- Logging advice `@Before` fires second
- Target method executes
- Logging advice `@After` fires first (inner unwinds first)
- Security advice `@After` fires last

---

## The five advice types — comparison

| Advice | Fires when | Receives | Can modify return | Can suppress exception |
|---|---|---|---|---|
| `@Before` | Before method | `JoinPoint` | ❌ | Only by throwing |
| `@After` | After always (finally) | `JoinPoint` | ❌ | ❌ |
| `@AfterReturning` | After normal return | `JoinPoint` + return value | ❌ | ❌ |
| `@AfterThrowing` | After exception | `JoinPoint` + exception | ❌ | ❌ |
| `@Around` | Wraps entire method | `ProceedingJoinPoint` | ✅ | ✅ |

---

## Plain-English summary

- `@Before` — runs before. Can't modify return value. Can prevent execution by throwing.
- `@After` — runs always (success or exception). No access to return value or exception.
- `@AfterReturning` — runs on normal return. Receives return value. Can't modify it.
- `@AfterThrowing` — runs on exception. Receives exception. Doesn't suppress it.
- `@Around` — wraps everything. Must call `proceed()` or target method is skipped. Must return a value. Only type that can modify return value or suppress exceptions.
- `ProceedingJoinPoint` is `@Around`-only. All other types use `JoinPoint`.
