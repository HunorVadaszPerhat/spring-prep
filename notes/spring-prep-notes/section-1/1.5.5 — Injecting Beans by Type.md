# Objective 1.5.5 — Injecting Beans by Type

## The problem this objective addresses

`@Autowired` resolves by type. As soon as you have two beans of the same type, Spring doesn't know which one to inject and throws `NoUniqueBeanDefinitionException`. This objective covers all the mechanisms Spring provides to resolve that ambiguity.

---

## `@Primary`

Marks one bean as the default choice when multiple candidates exist. Covered in 1.2.1 but worth restating in this context.

```java
@Configuration
public class PaymentConfig {

    @Bean
    @Primary
    public PaymentGateway stripeGateway() {
        return new StripePaymentGateway();
    }

    @Bean
    public PaymentGateway paypalGateway() {
        return new PaypalPaymentGateway();
    }
}

@Service
public class OrderService {
    @Autowired
    private PaymentGateway paymentGateway; // gets stripeGateway — it's @Primary
}
```

`@Primary` works at the bean definition level — one bean wins by default everywhere it's injected without a qualifier.

---

## `@Qualifier`

Specifies exactly which bean you want at the injection point. Overrides `@Primary`.

```java
@Service
public class OrderService {

    @Autowired
    @Qualifier("paypalGateway") // bean name as the qualifier
    private PaymentGateway paymentGateway;
}
```

You can also put `@Qualifier` on the bean definition itself to give it a qualifier name distinct from its bean name:

```java
@Bean
@Qualifier("international")
public PaymentGateway paypalGateway() {
    return new PaypalPaymentGateway();
}

// Then at injection point:
@Autowired
@Qualifier("international")
private PaymentGateway paymentGateway;
```

⚠️ **EXAM TRAP**: `@Qualifier` takes a **string name** — it's not type-safe. A typo compiles fine but fails at runtime with `NoSuchBeanDefinitionException`. This is one reason to prefer constructor injection with explicit wiring over field injection with qualifiers.

---

## `@Resource` (JSR-250)

`@Resource` is a Java standard annotation (`javax.annotation` / `jakarta.annotation`). It injects **by name first**, then falls back to type.

```java
@Service
public class OrderService {

    @Resource(name = "stripeGateway") // explicit name
    private PaymentGateway paymentGateway;

    @Resource // no name — uses field name "paypalGateway" as the lookup name
    private PaymentGateway paypalGateway;
}
```

Resolution order for `@Resource`:
1. By the explicit `name` attribute if provided
2. By the field/parameter name if no explicit name
3. By type as a last resort

⚠️ **EXAM TRAP**: `@Resource` is **name-first**, `@Autowired` is **type-first**. This is the key distinction the exam tests. If you have a field named `paypalGateway` and use `@Resource` without a name, Spring looks for a bean named `"paypalGateway"` — not a bean of type `PaymentGateway`.

---

## `@Inject` and `@Named` (JSR-330)

JSR-330 provides `@Inject` as a standard alternative to `@Autowired`, and `@Named` as a standard alternative to `@Qualifier`.

```java
import javax.inject.Inject;   // Boot 2.x
import javax.inject.Named;
// import jakarta.inject.Inject;  // Boot 3.x
// import jakarta.inject.Named;

@Service
public class OrderService {

    @Inject // behaves like @Autowired — resolves by type
    @Named("stripeGateway") // behaves like @Qualifier
    private PaymentGateway paymentGateway;
}
```

`@Inject` has no `required` attribute — there's no `@Inject(required = false)`. If the dependency isn't found, it always throws. Use `@Autowired(required = false)` or `Optional<T>` if you need optional injection.

---

## Field name as implicit qualifier — the fallback

As covered in 1.4.1, Spring uses the field name as a last-resort qualifier before throwing:

```java
@Autowired
private PaymentGateway stripeGateway; // if a bean named "stripeGateway" exists, it wins
```

This works but is fragile — renaming the field breaks the wiring silently. Prefer explicit `@Qualifier` over relying on field name matching.

---

## Injecting all beans of a type

When you genuinely want all implementations, inject a `List` or `Map`:

```java
@Service
public class PaymentRouter {

    @Autowired
    private List<PaymentGateway> gateways; // all PaymentGateway beans

    @Autowired
    private Map<String, PaymentGateway> gatewayMap; // bean name → bean instance
}
```

The `Map` form is particularly useful — the key is the bean name, so you can look up a specific gateway by name at runtime.

⚠️ **EXAM TRAP**: When injecting a `List<T>`, Spring injects **all** beans of type `T`. If you expected one bean but have two, you won't get `NoUniqueBeanDefinitionException` — you'll silently get both in a list. The exception only fires when injecting a single `T` with ambiguity.

---

## Resolution order — complete picture

When Spring resolves an `@Autowired` injection point with multiple candidates:

1. Find all beans matching the required type
2. If exactly one → done
3. Filter to `@Primary` candidates → if exactly one → done
4. Match against field/parameter **name** as implicit qualifier → if exactly one → done
5. Check for `@Qualifier` → if matches → done
6. Still ambiguous → throw `NoUniqueBeanDefinitionException`

---

## Comparison table

| Annotation | Standard | Resolves by | `required` support |
|---|---|---|---|
| `@Autowired` | Spring | Type first, name fallback | Yes (`required = false`) |
| `@Qualifier` | Spring | Name (used with `@Autowired`) | N/A |
| `@Resource` | JSR-250 | Name first, type fallback | No |
| `@Inject` | JSR-330 | Type first | No |
| `@Named` | JSR-330 | Name (used with `@Inject`) | N/A |

---

## Plain-English summary

- `@Primary` = default winner when ambiguous. Works everywhere the type is injected without a qualifier.
- `@Qualifier("name")` = explicit name at injection point. Overrides `@Primary`. String-based — typos fail at runtime.
- `@Resource` = JSR-250, name-first resolution. Field name used if no explicit name given.
- `@Inject` = JSR-330 equivalent of `@Autowired`. No `required` attribute.
- `@Named` = JSR-330 equivalent of `@Qualifier`.
- Inject `List<T>` to get all beans of a type. Inject `Map<String, T>` to get name → instance mapping.
- Key distinction: `@Autowired` is type-first. `@Resource` is name-first.
