# Objective 1.4.1 — `@Component`, `@ComponentScan`, `@Autowired`

## The two ways to define beans

You've seen the explicit way: write a `@Bean` method in a `@Configuration` class. The other way is **component scanning** — annotate your class directly and tell Spring to go find it. Spring scans packages, finds annotated classes, and registers them as beans automatically.

---

## `@Component` and `@ComponentScan`

`@Component` marks a class as a Spring-managed bean. `@ComponentScan` tells Spring which packages to search.

```java
@Component
public class PaymentGateway {
    // Spring finds this and registers it as a bean
    // Bean name defaults to the class name, lowercased: "paymentGateway"
}

@Configuration
@ComponentScan(basePackages = "com.example.payments")
public class AppConfig {
    // No @Bean methods needed — scanning handles it
}
```

With Spring Boot, `@SpringBootApplication` includes `@ComponentScan` that scans the package of the main class and all sub-packages. This is why Boot apps conventionally put the main class at the root package — everything below it gets scanned automatically.

⚠️ **EXAM TRAP**: `@ComponentScan` without `basePackages` scans the **package of the annotated class** and its sub-packages. If your components live in a sibling package that isn't a sub-package of the config class, they won't be found.

```
com.example.config.AppConfig       ← @ComponentScan here
com.example.services.OrderService  ← FOUND (sub-package of com.example)
com.other.payments.PaymentGateway  ← NOT FOUND (different root)
```

## `@Autowired`

`@Autowired` tells Spring to inject a dependency. It works on constructors, setters, and fields.

```java
@Component
public class OrderService {

    private final PaymentGateway paymentGateway;
    private final InventoryService inventoryService;

    // Constructor injection — @Autowired optional with single constructor (Spring 4.3+)
    @Autowired
    public OrderService(PaymentGateway paymentGateway,
                        InventoryService inventoryService) {
        this.paymentGateway = paymentGateway;
        this.inventoryService = inventoryService;
    }
}
```

Spring resolves `@Autowired` by **type** first. If exactly one bean of the required type exists, it's injected. If zero exist, it throws `NoSuchBeanDefinitionException`. If more than one exist, it throws `NoUniqueBeanDefinitionException` — unless you use `@Primary` or `@Qualifier` to resolve the ambiguity.

---

## Required vs optional injection

By default `@Autowired` is required — missing bean = startup failure. You can make it optional:

```java
@Autowired(required = false)
private MetricsCollector metricsCollector; // null if no bean exists, no error
```

Or use `Optional<T>`:

```java
@Autowired
private Optional<MetricsCollector> metricsCollector; // empty Optional if not found
```

⚠️ **EXAM TRAP**: `@Autowired(required = false)` on a constructor parameter means the whole constructor becomes optional injection — if any parameter can't be resolved, Spring skips the constructor entirely rather than partially injecting it. This is a subtle edge case the exam occasionally surfaces.

---

## How Spring resolves `@Autowired` step by step

1. Find all beans matching the required **type**
2. If exactly one → inject it
3. If multiple → check for `@Primary` → if found, inject it
4. If multiple and no `@Primary` → check if any bean **name** matches the field/parameter name
5. If still ambiguous → check for `@Qualifier`
6. If still ambiguous → throw `NoUniqueBeanDefinitionException`

Step 4 is worth knowing — Spring uses the **field name or parameter name** as a fallback qualifier before giving up.

```java
@Autowired
private PaymentGateway stripeGateway; // if a bean named "stripeGateway" exists, it wins
```

## `@ComponentScan` filtering

You can include or exclude classes from scanning:

```java
@ComponentScan(
    basePackages = "com.example",
    excludeFilters = @ComponentScan.Filter(
        type = FilterType.ANNOTATION,
        classes = Repository.class  // don't scan @Repository classes
    )
)
```

📝 **EXAM NOTE**: Know that filtering exists and that `FilterType.ANNOTATION` and `FilterType.ASSIGNABLE_TYPE` are the common filter types. Deep knowledge of all filter types isn't required.

---

## Plain-English summary

- `@Component` marks a class for auto-detection. Bean name = lowercased class name by default.
- `@ComponentScan` defines which packages to search. Without `basePackages`, scans the annotated class's own package and sub-packages.
- `@SpringBootApplication` includes `@ComponentScan` — scans from the main class's package downward.
- `@Autowired` injects by type. Ambiguity resolved by `@Primary`, then field name match, then `@Qualifier`.
- `@Autowired(required = false)` or `Optional<T>` for optional dependencies.
