# Objective 1.3.3 — Spring Expression Language (SpEL)

## What SpEL is for

SpEL is a runtime expression language built into Spring. It lets you embed dynamic expressions in annotations and XML config that are evaluated at runtime — not at compile time. The most common place you'll see it on the exam is inside `@Value`.

The key thing to hold onto: **`${...}` resolves property placeholders. `#{...}` evaluates SpEL expressions.** These look similar and the exam absolutely tests whether you can tell them apart.

---

## Basic `@Value` with SpEL

```java
@Component
public class AppStats {

    // Property placeholder — reads from application.properties
    @Value("${app.name}")
    private String appName;

    // SpEL — evaluates a Java expression at runtime
    @Value("#{2 * 60 * 1000}")
    private long timeoutMs; // evaluates to 120000

    @Value("#{T(java.lang.Math).PI}")
    private double pi; // accesses a static field via T() operator

    @Value("#{systemProperties['user.home']}")
    private String homeDir; // reads a JVM system property

    @Value("#{environment['spring.profiles.active']}")
    private String activeProfile; // reads from Spring Environment
}
```

`T()` is the **type operator** — it gives you access to a class's static methods and fields. `T(java.lang.Math).random()` would call `Math.random()` at bean creation time.

---

## Referencing other beans in SpEL

SpEL can reach into the container and reference other beans by name:

```java
@Component
public class ReportService {

    // References another bean and calls a method on it
    @Value("#{orderService.getDefaultPageSize()}")
    private int pageSize;

    // References a property on another bean
    @Value("#{paymentConfig.apiUrl}")
    private String paymentUrl;
}
```

The bean name goes directly inside `#{}`. Spring resolves it from the container at runtime.

---

## Combining SpEL with property placeholders

You can nest `${}` inside `#{}` — SpEL evaluates first, then the property is resolved:

```java
@Value("#{${payment.timeout:5000} * 2}")
private int doubledTimeout; // reads the property, then multiplies by 2
```

⚠️ **EXAM TRAP**: You cannot nest `#{}` inside `${}`. The outer expression must be SpEL (`#{}`) if you want to do computation. `${}` is a simple lookup with no expression capability.

---

## Common SpEL operators the exam tests

```java
@Value("#{10 > 5}")                    // boolean: true
@Value("#{user.age > 18 ? 'adult' : 'minor'}")  // ternary
@Value("#{someBean.value ?: 'default'}")         // Elvis operator — null-safe default
@Value("#{list.size()}")               // method call
@Value("#{list[0]}")                   // list index
@Value("#{map['key']}")                // map access
```

The **Elvis operator** (`?:`) is worth knowing — it's SpEL's shorthand for "if null, use this default". It's named after the Elvis hairstyle (the `?:` looks like a sideways face).

---

## Where SpEL is used beyond `@Value`

SpEL appears in other Spring annotations too. You don't need deep knowledge of these for 1.3.3 but you'll encounter them in later objectives:

- `@PreAuthorize("#{principal.name == 'admin'}")` — method security (Section 5)
- `@Cacheable(key = "#{#id}")` — cache key expressions
- `@ConditionalOnExpression` — conditional bean creation

---

## Plain-English summary

- `${key}` = property placeholder. Looks up a value from property sources. No computation.
- `#{expression}` = SpEL. Evaluates a dynamic expression at runtime. Can do math, call methods, reference beans.
- `T(ClassName)` accesses static members. `beanName.property` references another bean.
- Elvis operator `?:` = null-safe default. Ternary `? :` works as expected.
- Can nest `${}` inside `#{}` but not the other way around.
