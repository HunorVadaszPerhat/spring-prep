# Objective 1.6.3 — Pointcut Expressions

## What pointcut expressions do

A pointcut expression is the pattern that selects which method executions to intercept. Getting the syntax right is an exam skill — you need to read a pointcut and know what it matches, and spot a broken one.

---

## The `execution()` designator — the most important one

`execution()` matches method execution join points. It's the primary designator and the one the exam tests most heavily.

**Full syntax:**
```
execution(modifiers? return-type declaring-type?method-name(params) throws?)
```

Everything with `?` is optional. The minimum required is return type, method name, and parameters.

```java
// Read left to right: any return type, any class, any method, any args
execution(* *(..))  // matches everything — rarely useful

// Any return type, any method in OrderService, any args
execution(* com.example.service.OrderService.*(..))

// Any return type, any method in any class in the service package
execution(* com.example.service.*.*(..))

// Any return type, any method in service package OR sub-packages
execution(* com.example.service..*.*(..))  // note the double dot

// Specific return type, specific method, no args
execution(Order com.example.service.OrderService.findById(Long))

// void return type, method name starting with "process", any args
execution(void com.example.service.*.process*(..))
```

**Parameter patterns:**
- `()` — no parameters
- `(..)` — any number of any type (including zero)
- `(*)` — exactly one parameter of any type
- `(String, ..)` — first param is String, rest can be anything

⚠️ **EXAM TRAP**: `..` in the package position means "this package and all sub-packages". `..` in the parameter position means "any parameters". Same syntax, different meaning depending on context. The exam tests whether you can read these correctly.

---

## `within()` — match by type

Matches all method executions within a type or package, without specifying method details.

```java
// All methods in OrderService
within(com.example.service.OrderService)

// All methods in any class in the service package
within(com.example.service.*)

// All methods in service package and sub-packages
within(com.example.service..*)
```

The difference from `execution()`: `within()` is type-scoped — it doesn't let you filter by return type, method name, or parameters.

---

## `@annotation()` — match by annotation

Matches methods that carry a specific annotation. This is how Spring implements `@Transactional` interception.

```java
// Matches any method annotated with @Transactional
@annotation(org.springframework.transaction.annotation.Transactional)

// Matches any method annotated with your custom annotation
@annotation(com.example.annotation.Audited)
```

Practical use — apply advice to all methods marked with a custom annotation:

```java
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface Audited { }

@Aspect
@Component
public class AuditAspect {

    @Before("@annotation(com.example.annotation.Audited)")
    public void audit(JoinPoint jp) {
        System.out.println("Auditing: " + jp.getSignature().getName());
    }
}
```

---

## `bean()` — match by bean name

Spring-specific (not part of AspectJ). Matches methods on a specific named bean.

```java
// Matches all methods on the bean named "orderService"
bean(orderService)

// Wildcard — matches all beans whose name ends with "Service"
bean(*Service)
```

---

## Named pointcuts — reusing expressions

Instead of repeating the same expression in every advice method, declare a named pointcut:

```java
@Aspect
@Component
public class ServiceAspect {

    @Pointcut("execution(* com.example.service.*.*(..))")
    public void serviceLayer() { }  // method body is empty — just a named anchor

    @Before("serviceLayer()")  // reference by method name
    public void logBefore(JoinPoint jp) { ... }

    @After("serviceLayer()")   // reuse the same pointcut
    public void logAfter(JoinPoint jp) { ... }
}
```

The method annotated with `@Pointcut` must be `public`, return `void`, and have an empty body. It's just a named handle for the expression.

---

## Combining expressions with `&&`, `||`, `!`

```java
// Methods in service package AND annotated with @Audited
@Before("execution(* com.example.service.*.*(..)) && @annotation(com.example.annotation.Audited)")
public void auditServiceMethods(JoinPoint jp) { ... }

// Service layer OR repository layer
@Pointcut("within(com.example.service..*) || within(com.example.repository..*)")
public void serviceOrRepository() { }

// All service methods EXCEPT those annotated with @ReadOnly
@Before("serviceLayer() && !@annotation(com.example.annotation.ReadOnly)")
public void logWriteOperations(JoinPoint jp) { ... }
```

---

## Binding pointcut arguments to advice parameters

You can capture the annotation instance or method arguments directly in advice:

```java
// Capture the annotation itself
@Before("@annotation(audited)")  // lowercase — binds the annotation instance
public void audit(JoinPoint jp, Audited audited) {
    System.out.println("Audit level: " + audited.level());
}

// Capture a method argument by type
@Before("execution(* com.example.service.*.*(..)) && args(orderId, ..)")
public void logOrderId(JoinPoint jp, Long orderId) {
    System.out.println("Processing order: " + orderId);
}
```

⚠️ **EXAM TRAP**: When binding arguments, the name in the pointcut expression must match the parameter name in the advice method signature. Spring uses the name to wire them up.

---

## Common designators summary

| Designator | Matches on |
|---|---|
| `execution()` | Method signature — return type, class, name, params |
| `within()` | Type or package — all methods within |
| `@annotation()` | Methods carrying a specific annotation |
| `bean()` | Spring bean name (Spring-specific) |
| `args()` | Method arguments by type |
| `@within()` | Types carrying a specific annotation |
| `target()` | Target object type |

---

## Plain-English summary

- `execution()` is the primary designator. Pattern: `return-type class.method(params)`. `*` = any, `..` = any params or sub-packages.
- `within()` — all methods in a type or package. No method-level filtering.
- `@annotation()` — methods carrying a specific annotation. How `@Transactional` interception works.
- `bean()` — Spring-specific, matches by bean name. Supports wildcards.
- Named pointcuts with `@Pointcut` — declare once, reference by method name.
- Combine with `&&`, `||`, `!`.
- `..` in package position = sub-packages. `..` in param position = any arguments.
