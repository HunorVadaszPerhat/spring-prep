# Objective 4.2.4 — Slice Testing Patterns

## What slice tests are

A slice test loads only a specific layer of the application — just enough context to test one concern in isolation. This makes them faster than `@SpringBootTest` (no full context startup) and more focused (fewer beans to mock, clearer test intent).

Spring Boot provides several slice annotations. The exam focuses on three: `@WebMvcTest`, `@DataJpaTest`, and `@JsonTest`. `@WebMvcTest` and `@DataJpaTest` are covered in depth already — this objective consolidates and compares them, then adds `@JsonTest`.

---

## The three slices compared

| Annotation | What it loads | Auto-configures | Typical use |
|---|---|---|---|
| `@WebMvcTest` | Web layer — controllers, filters, advice | `MockMvc`, Jackson | Controller tests |
| `@DataJpaTest` | JPA layer — entities, repositories | H2, `TestEntityManager`, JPA | Repository tests |
| `@JsonTest` | Jackson serialisation only | `JacksonTester`, `ObjectMapper` | JSON serialisation tests |

Each slice is narrow and fast. Use `@SpringBootTest` only when you need multiple layers working together.

---

## `@WebMvcTest` — recap

```java
@WebMvcTest(OrderController.class)
class OrderControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean
    private OrderService orderService;  // not loaded by web slice — must mock

    @Test
    void shouldReturnOrder() throws Exception {
        when(orderService.findById(1L))
            .thenReturn(Optional.of(new Order(1L, "customer-1", BigDecimal.TEN)));

        mockMvc.perform(get("/orders/1"))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.customerName").value("customer-1"));
    }
}
```

Key point: specify the controller class in `@WebMvcTest(ControllerClass.class)` to load only that controller. Without specifying, all controllers are loaded — which may bring in more dependencies to mock.

## `@DataJpaTest` — recap

```java
@DataJpaTest
class OrderRepositoryTest {

    @Autowired
    private OrderRepository orderRepository;

    @Autowired
    private TestEntityManager entityManager;

    @Test
    void shouldFindByStatus() {
        entityManager.persistAndFlush(new Order("customer-1", BigDecimal.TEN));

        List<Order> pending = orderRepository.findByStatus("PENDING");
        assertThat(pending).hasSize(1);
    }
}
```

Key points recap:
- H2 in-memory by default — use `@AutoConfigureTestDatabase(replace = Replace.NONE)` for real DB
- `@Transactional` by default — each test rolls back
- No `@Service` or `@Controller` beans

## `@JsonTest` — serialisation testing

`@JsonTest` loads only Jackson configuration — the `ObjectMapper` and any `@JsonComponent` beans. Nothing else. Use it to verify that your objects serialise to and deserialise from JSON correctly, including custom serialisers, Jackson annotations, and field naming strategies.

```java
@JsonTest
class OrderJsonTest {

    @Autowired
    private JacksonTester<Order> json;  // typed helper for Order serialisation

    @Test
    void shouldSerialiseOrder() throws Exception {
        Order order = new Order(1L, "customer-1", BigDecimal.TEN);

        // serialise to JSON and assert structure
        JsonContent<Order> written = json.write(order);

        assertThat(written).extractingJsonPathNumberValue("$.id").isEqualTo(1);
        assertThat(written).extractingJsonPathStringValue("$.customerName")
            .isEqualTo("customer-1");
        assertThat(written).extractingJsonPathNumberValue("$.total")
            .isEqualTo(10);

        // assert field does NOT appear (e.g. @JsonIgnore)
        assertThat(written).doesNotHaveJsonPath("$.internalCode");
    }

    @Test
    void shouldDeserialiseOrder() throws Exception {
        String json = """
            {
                "id": 1,
                "customerName": "customer-1",
                "total": 10.00
            }
            """;

        // deserialise from JSON and assert fields
        Order order = this.json.parse(json).getObject();

        assertThat(order.getId()).isEqualTo(1L);
        assertThat(order.getCustomerName()).isEqualTo("customer-1");
        assertThat(order.getTotal()).isEqualByComparingTo(BigDecimal.TEN);
    }
}
```

`JacksonTester<T>` provides typed access to the Jackson serialisation pipeline. It respects all Jackson configuration — `@JsonIgnore`, `@JsonProperty`, custom serialisers, `ObjectMapper` customisations.

⚠️ **EXAM TRAP**: `@JsonTest` tests Jackson configuration only. It does not load the web layer, service layer, or any other beans. `JacksonTester` is the key class — it's not available outside of `@JsonTest` without manual setup.

## When to use which slice

```
Testing a controller method's HTTP behaviour?
→ @WebMvcTest

Testing a repository query or Spring Data method?
→ @DataJpaTest

Testing Jackson serialisation / custom @JsonComponent?
→ @JsonTest

Testing service layer logic with real dependencies?
→ @SpringBootTest(webEnvironment = NONE)

Testing the full HTTP stack end to end?
→ @SpringBootTest(webEnvironment = RANDOM_PORT)
```

## Adding beans to a slice

Sometimes a slice test needs a bean that isn't loaded by default. Three ways to add it:

**`@MockBean`** — mock it:
```java
@WebMvcTest(OrderController.class)
class OrderControllerTest {
    @MockBean
    private OrderService orderService;
}
```

**`@Import`** — add a real configuration class:
```java
@WebMvcTest(OrderController.class)
@Import(SecurityConfig.class)  // bring in security config not loaded by default
class OrderControllerTest { }
```

**`@TestConfiguration` nested class** — define test-only beans:
```java
@WebMvcTest(OrderController.class)
class OrderControllerTest {

    @TestConfiguration
    static class TestConfig {
        @Bean
        public OrderService orderService() {
            return new FakeOrderService();
        }
    }
}
```

## `@DataJpaTest` with a service dependency

If a repository somehow depends on a service (unusual but possible), `@DataJpaTest` won't load it. Options:

```java
@DataJpaTest
class OrderRepositoryTest {

    @MockBean
    private AuditService auditService;  // mock the dependency not in JPA slice

    @Autowired
    private OrderRepository orderRepository;
}
```

## Slice tests and `application.properties`

Slice tests load `application.properties` but auto-configuration is narrowed to the slice. This means:

- `@WebMvcTest` reads `server.port`, `spring.mvc.*` properties but ignores `spring.datasource.*`
- `@DataJpaTest` reads `spring.jpa.*` properties but ignores `spring.mvc.*`
- Profile-specific files load if `@ActiveProfiles` is used

---

## Plain-English summary

- Slice tests load only the relevant layer — faster and more focused than `@SpringBootTest`.
- `@WebMvcTest` — web layer. `MockMvc` auto-configured. Mock services with `@MockBean`.
- `@DataJpaTest` — JPA layer. H2 by default. `@Transactional` by default. `TestEntityManager` available.
- `@JsonTest` — Jackson only. `JacksonTester<T>` for serialisation/deserialisation assertions.
- Add missing beans to slices via `@MockBean`, `@Import`, or nested `@TestConfiguration`.
- Match the slice to what you're testing — don't reach for `@SpringBootTest` when a slice will do.
