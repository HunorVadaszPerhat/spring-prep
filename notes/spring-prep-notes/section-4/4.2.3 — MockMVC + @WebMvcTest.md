# Objective 4.2.3 — MockMVC + `@WebMvcTest`

## What `@WebMvcTest` loads

`@WebMvcTest` is a slice test annotation that loads only the **web layer**. It's faster than `@SpringBootTest` because it starts a minimal context — just enough to test your controllers.

What it loads:
- `@Controller`, `@RestController`, `@ControllerAdvice`, `@RestControllerAdvice`
- `@JsonComponent`
- `Filter` beans
- `WebMvcConfigurer` beans
- `MockMvc` auto-configured

What it does NOT load:
- `@Service` beans
- `@Repository` beans
- `@Component` beans outside the web layer
- JPA infrastructure
- Full auto-configuration

```java
@WebMvcTest(OrderController.class)  // load only OrderController's slice
class OrderControllerTest {

    @Autowired
    private MockMvc mockMvc;  // auto-configured — no @AutoConfigureMockMvc needed

    @MockBean
    private OrderService orderService;  // must mock — @Service not loaded

    @MockBean
    private OrderRepository orderRepository;  // must mock — @Repository not loaded
}
```

⚠️ **EXAM TRAP**: `@WebMvcTest` does not load `@Service` or `@Repository` beans. Any dependency your controller needs must be provided via `@MockBean`. Forgetting to mock a dependency causes `NoSuchBeanDefinitionException` at test startup.

## `MockMvc` — performing requests

`MockMvc` performs HTTP requests against the `DispatcherServlet` directly — no real HTTP connection, no server port. The full Spring MVC pipeline executes (handler mapping, handler adapter, message conversion, exception handling) but in-process.

**Basic request/response pattern:**

```java
@Test
void shouldReturnOrder() throws Exception {
    Order order = new Order(1L, "customer-1", BigDecimal.TEN);
    when(orderService.findById(1L)).thenReturn(Optional.of(order));

    mockMvc.perform(
            get("/orders/1")                          // build the request
                .accept(MediaType.APPLICATION_JSON)   // set Accept header
        )
        .andExpect(status().isOk())                   // assert status
        .andExpect(content().contentType(MediaType.APPLICATION_JSON))
        .andExpect(jsonPath("$.id").value(1))          // assert response body
        .andExpect(jsonPath("$.customerName").value("customer-1"));
}
```

## Building requests — `MockMvcRequestBuilders`

```java
// GET
mockMvc.perform(get("/orders/{id}", 1L))

// POST with JSON body
mockMvc.perform(
    post("/orders")
        .contentType(MediaType.APPLICATION_JSON)
        .content("{\"customerName\":\"customer-1\",\"total\":10.00}")
)

// POST with Jackson-serialised body (cleaner)
@Autowired
private ObjectMapper objectMapper;

mockMvc.perform(
    post("/orders")
        .contentType(MediaType.APPLICATION_JSON)
        .content(objectMapper.writeValueAsString(newOrder))
)

// PUT
mockMvc.perform(
    put("/orders/{id}", 1L)
        .contentType(MediaType.APPLICATION_JSON)
        .content(objectMapper.writeValueAsString(updatedOrder))
)

// DELETE
mockMvc.perform(delete("/orders/{id}", 1L))

// With query parameters
mockMvc.perform(get("/orders").param("status", "PENDING").param("page", "0"))

// With headers
mockMvc.perform(get("/orders").header("Authorization", "Bearer token"))
```

## Asserting responses — `MockMvcResultMatchers`

```java
// Status
.andExpect(status().isOk())           // 200
.andExpect(status().isCreated())      // 201
.andExpect(status().isNoContent())    // 204
.andExpect(status().isNotFound())     // 404
.andExpect(status().isBadRequest())   // 400
.andExpect(status().is(HttpStatus.CONFLICT.value()))  // any status

// Content type
.andExpect(content().contentType(MediaType.APPLICATION_JSON))

// Response body — string
.andExpect(content().string("expected body"))

// Response body — JSON path
.andExpect(jsonPath("$.id").value(1))
.andExpect(jsonPath("$.customerName").value("customer-1"))
.andExpect(jsonPath("$.items").isArray())
.andExpect(jsonPath("$.items").isEmpty())
.andExpect(jsonPath("$.items[0].name").value("Widget"))
.andExpect(jsonPath("$").isNotEmpty())

// Headers
.andExpect(header().string("Location", "/orders/1"))
.andExpect(header().exists("X-Order-Id"))
```

## Debugging — `andDo(print())`

```java
mockMvc.perform(get("/orders/1"))
    .andDo(print())  // prints full request and response to console
    .andExpect(status().isOk());
```

Invaluable when a test fails unexpectedly — prints the request, response status, headers, and body. Remove before committing.

## Testing exception handling

`@WebMvcTest` includes `@RestControllerAdvice` beans — global exception handling is active:

```java
@Test
void shouldReturn404WhenOrderNotFound() throws Exception {
    when(orderService.findById(999L)).thenReturn(Optional.empty());
    // or throw an exception:
    when(orderService.findById(999L)).thenThrow(new OrderNotFoundException(999L));

    mockMvc.perform(get("/orders/999"))
        .andExpect(status().isNotFound())
        .andExpect(jsonPath("$.message").value("Order not found: 999"));
}

@Test
void shouldReturn400WhenOrderInvalid() throws Exception {
    Order invalid = new Order(null, null);  // fails @Valid

    mockMvc.perform(
            post("/orders")
                .contentType(MediaType.APPLICATION_JSON)
                .content(objectMapper.writeValueAsString(invalid))
        )
        .andExpect(status().isBadRequest());
}
```

## `@WithMockUser` — security context in MockMvc tests

When Spring Security is on the classpath, `@WebMvcTest` includes the security filter chain. Unauthenticated requests to secured endpoints return 401 or redirect to login.

`@WithMockUser` sets up a mock security context for the test:

```java
@Test
@WithMockUser(username = "alice", roles = {"USER"})  // mock authenticated user
void shouldAllowAuthenticatedAccess() throws Exception {
    mockMvc.perform(get("/orders"))
        .andExpect(status().isOk());
}

@Test
@WithMockUser(username = "admin", roles = {"ADMIN"})
void shouldAllowAdminAccess() throws Exception {
    mockMvc.perform(delete("/orders/1"))
        .andExpect(status().isNoContent());
}

@Test
void shouldReturn401WhenNotAuthenticated() throws Exception {
    mockMvc.perform(get("/orders"))
        .andExpect(status().isUnauthorized());
}
```

📝 **EXAM NOTE**: `@WithMockUser` is a good-to-know annotation for the exam — it appears in questions about testing secured endpoints but isn't a primary objective. Know it exists and what it does.

## `@WebMvcTest` vs `@SpringBootTest(MOCK)` with `@AutoConfigureMockMvc`

Both give you `MockMvc`. The difference is the context size:

| | `@WebMvcTest` | `@SpringBootTest(MOCK)` + `@AutoConfigureMockMvc` |
|---|---|---|
| Context size | Web layer only | Full application context |
| Speed | Faster | Slower |
| Service beans available | No — must `@MockBean` | Yes — real beans |
| Use for | Isolated controller tests | Full integration with real services |

Use `@WebMvcTest` when you want to test the controller in isolation. Use `@SpringBootTest` when you need real services wired together.

---

## Plain-English summary

- `@WebMvcTest` loads web layer only — controllers, advice, filters. No services or repositories.
- Missing dependencies must be provided via `@MockBean`.
- `MockMvc` executes the full MVC pipeline in-process — no real HTTP.
- `perform()` builds the request. `andExpect()` asserts. `andDo(print())` debugs.
- `jsonPath()` for asserting JSON response body. Uses dot notation for nested properties.
- `@RestControllerAdvice` is active — exception handling works in `@WebMvcTest`.
- `@WithMockUser` mocks a security context for secured endpoint tests.
- `@WebMvcTest` faster than `@SpringBootTest` — use for isolated controller tests.
