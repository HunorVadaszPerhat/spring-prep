# Objective 3.1.3 — `@RestController`, `@GetMapping`, `ResponseEntity`

## `@RestController`

As covered in 1.4.4, `@RestController` is a composed annotation:

```java
@RestController = @Controller + @ResponseBody
```

`@Controller` marks the class as a Spring MVC handler and registers it for component scanning. `@ResponseBody` tells Spring that every method's return value should be written directly to the HTTP response body via `HttpMessageConverter` — not interpreted as a view name.

```java
@RestController
@RequestMapping("/orders")  // base path for all methods in this class
public class OrderController {
    // all methods return data serialised to JSON — no view resolution
}
```

Without `@ResponseBody` (i.e. plain `@Controller`), returning a `String` from a method means "render this view template". With `@ResponseBody`, returning a `String` means "write this string to the response body".

⚠️ **EXAM TRAP**: `@RestController` is a composed annotation — `@Controller` + `@ResponseBody`. If the exam asks what `@RestController` is equivalent to, the answer requires both annotations.

## Request mapping annotations

`@RequestMapping` is the base annotation. The HTTP-method-specific variants are shortcuts:

```java
@GetMapping("/orders")          // = @RequestMapping(method = RequestMethod.GET)
@PostMapping("/orders")         // = @RequestMapping(method = RequestMethod.POST)
@PutMapping("/orders/{id}")     // = @RequestMapping(method = RequestMethod.PUT)
@DeleteMapping("/orders/{id}")  // = @RequestMapping(method = RequestMethod.DELETE)
@PatchMapping("/orders/{id}")   // = @RequestMapping(method = RequestMethod.PATCH)
```

`@RequestMapping` at the class level sets a base path. Method-level mappings append to it:

```java
@RestController
@RequestMapping("/orders")  // base path
public class OrderController {

    @GetMapping            // GET /orders
    public List<Order> findAll() { ... }

    @GetMapping("/{id}")   // GET /orders/{id}
    public Order findById(@PathVariable Long id) { ... }

    @PostMapping           // POST /orders
    public Order create(@RequestBody Order order) { ... }
}
```

## Parameter annotations

```java
@GetMapping("/{id}")
public Order findById(
    @PathVariable Long id) { }          // from URL path: /orders/123

@GetMapping
public List<Order> findAll(
    @RequestParam String status,         // from query string: ?status=PENDING
    @RequestParam(required = false,
                  defaultValue = "0")
    int page) { }                        // optional with default

@PostMapping
public Order create(
    @RequestBody Order order) { }        // from request body — deserialised by Jackson

@GetMapping("/search")
public List<Order> search(
    @RequestHeader("X-Customer-Id")
    String customerId) { }               // from HTTP header
```

⚠️ **EXAM TRAP**: `@RequestParam` is **required by default**. If the parameter is missing from the query string and no `defaultValue` is set, Spring throws `MissingServletRequestParameterException` (400 Bad Request). Set `required = false` or provide a `defaultValue` to make it optional.

## `ResponseEntity<T>`

`ResponseEntity` gives you full control over the HTTP response — status code, headers, and body — all in one return value.

```java
@GetMapping("/{id}")
public ResponseEntity<Order> findById(@PathVariable Long id) {
    return orderService.findById(id)
        .map(order -> ResponseEntity.ok(order))          // 200 OK with body
        .orElse(ResponseEntity.notFound().build());      // 404 Not Found, no body
}
```

**Builder methods:**
```java
ResponseEntity.ok(body)                    // 200 OK
ResponseEntity.ok().build()                // 200 OK, no body
ResponseEntity.created(uri).body(order)    // 201 Created with Location header
ResponseEntity.noContent().build()         // 204 No Content
ResponseEntity.notFound().build()          // 404 Not Found
ResponseEntity.badRequest().build()        // 400 Bad Request
ResponseEntity.status(HttpStatus.CONFLICT).body(error)  // any status with body
```

**Setting headers:**
```java
@PostMapping
public ResponseEntity<Order> create(@RequestBody Order order) {
    Order saved = orderService.save(order);
    URI location = URI.create("/orders/" + saved.getId());
    return ResponseEntity
        .created(location)              // 201 with Location header
        .header("X-Order-Id", saved.getId().toString())  // custom header
        .body(saved);
}
```

## When to use `ResponseEntity` vs plain return type

**Plain return type** — Spring infers 200 OK, serialises the body:
```java
@GetMapping("/{id}")
public Order findById(@PathVariable Long id) {
    return orderService.findById(id);  // always 200 OK
}
```

**`ResponseEntity`** — when you need control over status code, headers, or conditional responses:
```java
@GetMapping("/{id}")
public ResponseEntity<Order> findById(@PathVariable Long id) {
    return orderService.findById(id)
        .map(ResponseEntity::ok)
        .orElse(ResponseEntity.notFound().build());  // 404 when not found
}
```

⚠️ **EXAM TRAP**: A plain return type always produces 200 OK — you can't return 404 without `ResponseEntity` or throwing an exception handled by `@ExceptionHandler`.

---

## `@ResponseStatus`

Simpler alternative to `ResponseEntity` when you just need a fixed status code:

```java
@PostMapping
@ResponseStatus(HttpStatus.CREATED)  // always 201, no need for ResponseEntity
public Order create(@RequestBody Order order) {
    return orderService.save(order);
}

@DeleteMapping("/{id}")
@ResponseStatus(HttpStatus.NO_CONTENT)  // 204
public void delete(@PathVariable Long id) {
    orderService.delete(id);
    // void method + @ResponseStatus(NO_CONTENT) = 204 with no body
}
```

`@ResponseStatus` on the method sets a fixed status. It can't conditionally return different status codes — for that you need `ResponseEntity`.

---

## Content negotiation

Spring picks the serialisation format based on the `Accept` header in the request:

```
Accept: application/json  →  MappingJackson2HttpMessageConverter (JSON)
Accept: application/xml   →  Jaxb2RootElementHttpMessageConverter (XML)
```

With Boot and Jackson on the classpath, JSON is the default. XML requires adding `jackson-dataformat-xml`.

---

## Plain-English summary

- `@RestController` = `@Controller` + `@ResponseBody`. All methods write to response body, no view resolution.
- `@GetMapping`, `@PostMapping`, etc. are shortcuts for `@RequestMapping(method = ...)`.
- `@PathVariable` — from URL path. `@RequestParam` — from query string (required by default). `@RequestBody` — deserialised from request body.
- `ResponseEntity` — full control over status code, headers, and body. Use when response varies.
- Plain return type = always 200 OK. Use `ResponseEntity` or `@ResponseStatus` for other codes.
- `@ResponseStatus` on method = fixed status code. Can't conditionally vary it.
