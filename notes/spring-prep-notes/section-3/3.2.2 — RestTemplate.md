# Objective 3.2.2 — `RestTemplate`

## What `RestTemplate` is

`RestTemplate` is Spring's synchronous HTTP client for making REST API calls from your application to other services. It handles the mechanics of opening connections, serialising request bodies, deserialising responses, and translating HTTP errors into Spring exceptions.

📝 **EXAM NOTE**: The exam tests `RestTemplate` only. `WebClient` (the reactive, non-blocking alternative introduced in Spring 5) is explicitly out of scope. If you see `WebClient` in an answer option, it's a distractor.

---

## Creating a `RestTemplate`

With Spring Boot, `RestTemplate` is **not** auto-configured as a bean — you must declare it yourself. The recommended approach is via `RestTemplateBuilder`, which Boot does auto-configure:

```java
@Configuration
public class HttpClientConfig {

    @Bean
    public RestTemplate restTemplate(RestTemplateBuilder builder) {
        return builder
            .connectTimeout(Duration.ofSeconds(5))
            .readTimeout(Duration.ofSeconds(10))
            .build();
    }
}
```

`RestTemplateBuilder` applies Boot's auto-configured defaults (message converters, interceptors) before building. Using `new RestTemplate()` directly bypasses these defaults.

⚠️ **EXAM TRAP**: `RestTemplate` is not auto-configured as a bean — you must declare it. `RestTemplateBuilder` IS auto-configured and should be used to build it. The exam tests this distinction.

## The core methods

### `getForObject()` — GET, deserialise body directly

```java
// Returns the response body deserialised to the specified type
Order order = restTemplate.getForObject(
    "https://api.example.com/orders/{id}",
    Order.class,    // response body type
    123L            // URI variable — replaces {id}
);
// Returns null if response body is empty
```

Simple but limited — you only get the body, no access to status code or headers.

---

### `getForEntity()` — GET, returns full `ResponseEntity`

```java
// Returns ResponseEntity — body + status + headers
ResponseEntity<Order> response = restTemplate.getForEntity(
    "https://api.example.com/orders/{id}",
    Order.class,
    123L
);

HttpStatus status = response.getStatusCode();
Order order = response.getBody();
HttpHeaders headers = response.getHeaders();
```

Use when you need more than just the body — status code checking, reading response headers.

---

### `postForEntity()` — POST, returns `ResponseEntity`

```java
Order newOrder = new Order("customer-1", BigDecimal.TEN);

ResponseEntity<Order> response = restTemplate.postForEntity(
    "https://api.example.com/orders",
    newOrder,       // request body — serialised to JSON
    Order.class     // response body type
);

Order created = response.getBody();
URI location = response.getHeaders().getLocation(); // Location header from 201 response
```

---

### `postForObject()` — POST, returns body directly

```java
Order created = restTemplate.postForObject(
    "https://api.example.com/orders",
    newOrder,
    Order.class
);
```

Same trade-off as `getForObject` vs `getForEntity` — simpler but no access to status or headers.

---

### `exchange()` — full control, any HTTP method

`exchange()` is the most flexible method. Use it when you need to set request headers, use HTTP methods not covered by dedicated methods (`PUT`, `DELETE`, `PATCH`), or need the full `ResponseEntity` response.

```java
// GET with custom headers
HttpHeaders headers = new HttpHeaders();
headers.set("Authorization", "Bearer " + token);
headers.setAccept(List.of(MediaType.APPLICATION_JSON));

HttpEntity<Void> requestEntity = new HttpEntity<>(headers); // no body for GET

ResponseEntity<Order> response = restTemplate.exchange(
    "https://api.example.com/orders/{id}",
    HttpMethod.GET,
    requestEntity,
    Order.class,
    123L
);

// PUT with body and headers
Order updatedOrder = new Order("customer-1", BigDecimal.TEN);
HttpEntity<Order> putEntity = new HttpEntity<>(updatedOrder, headers);

ResponseEntity<Order> putResponse = restTemplate.exchange(
    "https://api.example.com/orders/{id}",
    HttpMethod.PUT,
    putEntity,
    Order.class,
    123L
);

// DELETE
ResponseEntity<Void> deleteResponse = restTemplate.exchange(
    "https://api.example.com/orders/{id}",
    HttpMethod.DELETE,
    new HttpEntity<>(headers),
    Void.class,
    123L
);
```

`HttpEntity<T>` wraps the request body and headers together. For requests with no body (GET, DELETE), use `HttpEntity<Void>` or `HttpEntity.EMPTY`.

## URI variables — two styles

**Positional varargs:**
```java
restTemplate.getForObject(
    "https://api.example.com/orders/{id}/items/{itemId}",
    Item.class,
    123L,   // {id}
    456L    // {itemId}
);
```

**Map:**
```java
Map<String, Object> vars = Map.of("id", 123L, "itemId", 456L);
restTemplate.getForObject(
    "https://api.example.com/orders/{id}/items/{itemId}",
    Item.class,
    vars
);
```

## Error handling

By default, `RestTemplate` throws `HttpClientErrorException` for 4xx responses and `HttpServerErrorException` for 5xx responses. Both extend `RestClientResponseException` which extends `RuntimeException`.

```java
try {
    Order order = restTemplate.getForObject(url, Order.class, id);
} catch (HttpClientErrorException.NotFound e) {
    // 404 Not Found
    return Optional.empty();
} catch (HttpClientErrorException e) {
    // other 4xx errors
    log.error("Client error: {} {}", e.getStatusCode(), e.getResponseBodyAsString());
    throw e;
} catch (HttpServerErrorException e) {
    // 5xx errors
    log.error("Server error: {}", e.getStatusCode());
    throw e;
}
```

Common `HttpClientErrorException` subclasses:
- `HttpClientErrorException.NotFound` — 404
- `HttpClientErrorException.BadRequest` — 400
- `HttpClientErrorException.Unauthorized` — 401
- `HttpClientErrorException.Forbidden` — 403

## Deserialising generic types — `ParameterizedTypeReference`

`getForObject(url, List.class)` gives you a raw `List` — Jackson can't infer the element type at runtime due to type erasure. Use `ParameterizedTypeReference` with `exchange()` to preserve generic type information:

```java
// Wrong — returns List<LinkedHashMap>, not List<Order>
List<Order> orders = restTemplate.getForObject(url, List.class);

// Correct — preserves generic type information
ResponseEntity<List<Order>> response = restTemplate.exchange(
    "https://api.example.com/orders",
    HttpMethod.GET,
    null,
    new ParameterizedTypeReference<List<Order>>() {}  // anonymous subclass captures type
);
List<Order> orders = response.getBody();
```

⚠️ **EXAM TRAP**: `getForObject(url, List.class)` does not correctly deserialise a JSON array into `List<Order>`. You must use `exchange()` with `ParameterizedTypeReference` to deserialise generic types correctly.

## Method summary

| Method | HTTP | Returns | Use when |
|---|---|---|---|
| `getForObject()` | GET | Body only | Simple GET, don't need status/headers |
| `getForEntity()` | GET | `ResponseEntity` | GET + need status or headers |
| `postForObject()` | POST | Body only | Simple POST, don't need status/headers |
| `postForEntity()` | POST | `ResponseEntity` | POST + need status or headers |
| `exchange()` | Any | `ResponseEntity` | Custom headers, PUT/DELETE/PATCH, generic types |
| `execute()` | Any | Anything | Low-level, full callback control (rarely needed) |

---

## Plain-English summary

- `RestTemplate` is not auto-configured — declare it as a `@Bean` using `RestTemplateBuilder`.
- `getForObject()` / `postForObject()` — return body directly. Simpler but no status/header access.
- `getForEntity()` / `postForEntity()` — return `ResponseEntity`. Full status and header access.
- `exchange()` — full control. Any HTTP method, custom headers, generic response types.
- `HttpEntity<T>` wraps request body + headers for `exchange()`.
- 4xx → `HttpClientErrorException`. 5xx → `HttpServerErrorException`. Both unchecked.
- `ParameterizedTypeReference` with `exchange()` for correct generic type deserialisation.
