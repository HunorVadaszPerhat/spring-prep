# Objective 6.1.2 — Spring Boot Dependency Management

## The problem Spring Boot solves

Managing library versions in a Java project is painful. Jackson, Hibernate, Tomcat, Spring Framework itself — they all have their own release cycles and compatibility constraints. Getting the wrong combination causes runtime failures that are hard to diagnose.

Spring Boot solves this with a curated **Bill of Materials (BOM)** — a list of dependency versions that are tested and known to work together. You declare a dependency without a version number and Spring Boot's BOM supplies the version.

---

## `spring-boot-starter-parent`

The standard way to use Spring Boot in a Maven project. Your `pom.xml` declares it as the parent:

```xml
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>2.5.14</version>  <!-- the only version you manage manually -->
</parent>
```

`spring-boot-starter-parent` gives you:
- Inherits from `spring-boot-dependencies` — the actual BOM with all version numbers
- Default Java version configuration
- Default encoding (UTF-8)
- Sensible Maven plugin defaults (`maven-compiler-plugin`, `spring-boot-maven-plugin`)
- Resource filtering configuration

⚠️ **EXAM TRAP**: `spring-boot-starter-parent` is the parent POM your project inherits from. `spring-boot-dependencies` is the BOM it inherits from — this is where the actual version numbers live. The exam tests whether you know this two-level hierarchy.

---

## `spring-boot-dependencies` BOM — without parent POM

If your project already has a corporate parent POM and can't use `spring-boot-starter-parent`, import the BOM directly:

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-dependencies</artifactId>
            <version>2.5.14</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

This imports all the managed versions without the parent POM relationship. You don't get the plugin defaults or resource filtering — just the version management.

---

## Starter POMs — bundled dependency sets

Starters are convenience POMs that pull in a curated set of dependencies for a specific feature area. They exist so you don't have to know every individual artifact needed for a feature.

```xml
<!-- Web — includes Spring MVC, Tomcat, Jackson -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <!-- no version — managed by parent/BOM -->
</dependency>

<!-- Data JPA — includes Hibernate, Spring Data JPA, JDBC -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>

<!-- Security -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>

<!-- Testing -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
```

No version numbers — the BOM manages them all.

---

## What each starter includes — exam-tested content

The exam asks what a specific starter pulls in. Know these:

**`spring-boot-starter-web`:**
- Spring MVC
- Embedded Tomcat
- Jackson (JSON serialisation)
- `spring-boot-starter` (core)
- `spring-boot-starter-json`
- Hibernate Validator (for `@Valid`)

**`spring-boot-starter-data-jpa`:**
- Hibernate (JPA implementation)
- Spring Data JPA
- Spring JDBC
- `spring-boot-starter-jdbc`
- Agroal connection pool (or HikariCP in Boot 2.x)

**`spring-boot-starter-test`:**
- JUnit 5
- Mockito
- AssertJ
- Hamcrest
- JSONassert
- JsonPath
- Spring Test + Spring Boot Test

**`spring-boot-starter-aop`:**
- Spring AOP
- AspectJ weaver

⚠️ **EXAM TRAP**: `spring-boot-starter-web` includes **Tomcat** — that's why you have to explicitly exclude it to switch to Jetty or Undertow. The exam may ask what's included in the web starter or why switching servers requires an exclusion.

## Overriding a managed version

Sometimes you need a different version than what Boot manages — a security patch, a newer feature. Two ways:

**Via property** (preferred — `spring-boot-starter-parent` exposes version properties):

```xml
<properties>
    <jackson.version>2.14.0</jackson.version>    <!-- override Jackson version -->
    <hibernate.version>5.6.15.Final</hibernate.version>
</properties>
```

Each managed dependency has a corresponding property. Check the Spring Boot documentation for the property name — it follows the pattern `{library}.version`.

**Via explicit version in `<dependency>`:**

```xml
<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-databind</artifactId>
    <version>2.14.0</version>  <!-- explicit version overrides BOM -->
</dependency>
```

⚠️ **EXAM TRAP**: Overriding a managed version can break compatibility. Spring Boot's BOM versions are tested together — changing one library's version without changing compatible libraries can cause subtle runtime failures. The exam tests that you know overriding is possible but carries risk.

---

## Adding non-managed dependencies

For libraries not in the BOM, you must provide the version yourself:

```java
<dependency>
    <groupId>com.google.guava</groupId>
    <artifactId>guava</artifactId>
    <version>31.1-jre</version>  <!-- not in BOM — version required -->
</dependency>
```

## The naming convention for starters

Official Spring Boot starters follow the pattern `spring-boot-starter-{feature}`. Third-party starters conventionally use `{name}-spring-boot-starter` to distinguish them:

```
spring-boot-starter-web          ← official Spring Boot starter
spring-boot-starter-data-jpa     ← official Spring Boot starter
mybatis-spring-boot-starter      ← third-party starter (MyBatis)
```

⚠️ **EXAM TRAP**: The naming convention distinction — official starters start with `spring-boot-starter-`, third-party starters end with `-spring-boot-starter`. The exam may present options and ask which is the official Spring Boot starter.

## Plain-English summary

- `spring-boot-starter-parent` — parent POM. Inherits from `spring-boot-dependencies` BOM. Provides plugin defaults and version management.
- `spring-boot-dependencies` — the actual BOM. Import directly if you can't use the parent POM.
- Starters — curated dependency bundles. No version numbers needed — BOM manages them.
- `spring-boot-starter-web` includes Spring MVC, Tomcat, Jackson, Hibernate Validator.
- `spring-boot-starter-data-jpa` includes Hibernate, Spring Data JPA, HikariCP.
- `spring-boot-starter-test` includes JUnit 5, Mockito, AssertJ, Hamcrest, JSONassert, JsonPath.
- Override managed versions via properties (`<jackson.version>`) or explicit version in `<dependency>`.
- Official starters: `spring-boot-starter-{feature}`. Third-party: `{name}-spring-boot-starter`.
